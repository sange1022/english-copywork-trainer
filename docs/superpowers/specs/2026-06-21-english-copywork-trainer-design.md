# 英文打字临摹练习器设计

## 目标

制作一个可在本机运行的网页应用。用户粘贴一段英文后，应用自动分句并获取中文翻译，再通过逐字键盘输入完成临摹练习，以加强英文拼写、句型和输入记忆。

第一版不要求注册、登录或配置 API Key。除免费翻译与单词释义需要联网外，练习页面和历史记录均在本机运行和保存。

## 使用流程

1. 用户双击启动脚本。
2. 本地服务启动，并自动在默认浏览器中打开应用。
3. 用户将一段或多段英文粘贴到首页输入框。
4. 用户点击“开始练习”。
5. 应用按英文句末标点自动分句，并请求每句的中文翻译。
6. 第一条练习依次显示英文原文、中文翻译和淡灰色的英文临摹文本。
7. 用户使用键盘逐字输入。
8. 当前句全部输入正确后，应用等待约 1 秒，再自动进入下一句。
9. 全部句子完成后，显示本次练习结果。

## 页面与交互

### 输入页

页面顶部提供一个较大的多行输入框，用于粘贴英文内容。输入框下方显示：

- “开始练习”按钮
- 最近练习记录入口
- 简短说明：翻译功能需要联网，练习内容保存在本机

空内容不能开始练习，并显示明确提示。

### 练习页

练习页按照固定顺序显示：

1. 可点击单词的英文原句
2. 当前句的中文翻译
3. 淡灰色英文临摹行
4. 进度、正确率和用时

临摹行中的目标句作为浅色底字显示，用户输入与目标文字在同一位置对照：

- 已正确输入的字符显示为深色。
- 当前错误字符显示为红色。
- 出错后保留错误字符和红色提示，用户必须退格修改。
- 错误修正前，不允许继续推进匹配位置。
- 空格、大小写和英文标点均参与严格匹配。
- 完成当前句后锁定输入，显示完成反馈，并在约 1 秒后进入下一句。

页面提供“退出练习”操作。退出前要求确认，避免误触丢失本次进度。

### 单词释义

英文原句按单词切分为可点击元素。点击单词后显示小型释义浮层，包括：

- 当前单词
- 中文释义
- 加载状态
- 查询失败后的重试提示

查询时去除单词外围标点，但保留页面中的原始句子。相同单词的查询结果缓存在本机，减少重复网络请求。

### 完成页

全部句子完成后显示：

- 完成句数
- 总字符数
- 错误按键次数
- 正确率
- 总用时
- “再练一次”和“返回首页”按钮

“再练一次”复用当前文章和翻译，不重复请求已有翻译。

## 文本处理规则

### 自动分句

应用以 `.`, `!`, `?` 及其连续组合为主要句末标记，并兼容英文引号或括号跟在句末标点之后的情况。换行和多余空白会被规范化为空格。

分句结果应满足：

- 不产生空句。
- 保留句末标点。
- 不改变原文中的字母大小写。
- 无句末标点的最后一段仍作为一句。

第一版不专门处理所有英文缩写歧义；对于 `Mr.`、`Dr.`、`e.g.` 等常见缩写，使用一份小型例外表避免明显误分。

### 输入匹配

内部使用目标字符串、已输入字符串和当前索引进行匹配。浏览器输入事件需要兼容粘贴、退格和输入法组合事件，但练习区禁止直接粘贴答案。

错误次数按错误按键计数；删除错误字符不会撤销已经产生的错误记录。正确率计算为：

`目标字符总数 / (目标字符总数 + 错误按键次数)`

## 翻译策略

本地服务负责调用无需用户配置 API Key 的免费在线翻译服务，避免浏览器跨域限制并隐藏具体供应方细节。

翻译层采用独立适配器，页面只调用本地统一接口：

- 句子翻译接口：英文句子转换为中文。
- 单词释义接口：英文单词转换为简短中文释义。

为降低免费接口不稳定带来的影响：

- 翻译结果写入本地缓存。
- 请求设置超时和有限次数重试。
- 某一句翻译失败时，仍允许进入练习，并在中文位置显示“翻译暂不可用”。
- 单词查询失败不影响打字练习。
- 服务供应方以后可以替换，不改变前端练习逻辑。

免费翻译服务的可用性不作永久保证，因此失败状态必须是正常、可恢复的页面状态。

## 本地数据

第一版保存以下内容：

- 原始英文文章
- 自动分句结果
- 已获取的句子翻译
- 单词释义缓存
- 每次练习的完成时间、用时、正确率和错误次数

数据只保存在本机。练习记录设置合理上限，超出后删除最旧记录，避免无限增长。

## 技术结构

应用由三个边界清晰的部分组成：

### 浏览器界面

负责输入页、练习页、完成页、逐字匹配、动画反馈和本地记录展示。界面应适配常见桌面浏览器宽度，第一版以键盘和电脑使用为主。

### 本地服务

负责提供静态网页、转发翻译请求、缓存翻译结果，并在启动时打开浏览器。服务仅监听本机回环地址，不对局域网公开。

### 启动入口

提供 macOS 可双击运行的启动脚本。脚本检查运行环境、启动服务并打开页面；重复启动时复用现有服务或给出清晰提示。

## 错误处理

- 启动环境缺失：在终端窗口显示中文安装提示。
- 本地端口占用：自动尝试备用端口。
- 翻译超时或失败：显示降级文案，允许继续练习。
- 输入内容为空：停留在输入页并提示。
- 文本过长：第一版设置明确长度上限，并提示用户分批练习。
- 本地记录损坏：跳过损坏记录，不阻止应用启动。

## 测试重点

### 自动化测试

- 普通英文段落、连续标点、引号、换行和常见缩写的分句结果
- 正确字符、错误字符、退格修正、空格、大小写和标点匹配
- 错误次数及正确率计算
- 翻译成功、超时、重试、失败和缓存命中
- 文章与练习记录的保存和读取

### 浏览器交互测试

- 从粘贴文章到完成练习的完整流程
- 错误字符变红且修正前无法推进
- 完成一句后约 1 秒自动进入下一句
- 点击单词显示释义
- 翻译服务不可用时仍可完成练习
- 双击启动后自动打开网页

## 第一版范围

包含：

- 英文粘贴输入
- 自动分句
- 自动中文翻译
- 逐字严格临摹
- 错误红色反馈
- 完成后自动切换下一句
- 点击单词查询释义
- 练习统计和本地历史
- macOS 双击启动

暂不包含：

- 用户账户或云同步
- AI 对话和写作评分
- 发音录音或语音识别
- 自定义词库和记忆曲线
- 手机端专门适配
- Windows 安装包

## 验收标准

在一台具备网络连接和支持环境的 Mac 上，用户可以通过双击启动应用，粘贴英文并开始练习。应用能自动分句和展示中文翻译；用户必须按顺序正确输入每个字符，错误字符会变红，修正后才能继续。当前句完成约 1 秒后自动进入下一句。点击原文单词可查看中文释义。即使翻译服务暂时失败，用户仍能完成整篇打字练习并看到练习结果。

## 2026-06-22 中文练习扩展

### 自动模式识别

应用按整篇输入内容识别练习模式：

- 内容包含汉字时使用中文模式。
- 不含汉字时使用英文模式。
- 中英混合内容按中文模式处理。

### 中文模式

- 按中文句末标点 `。`、`！`、`？` 自动分句，同时兼容英文句末标点。
- 不请求或显示翻译，也不显示“重新翻译”按钮。
- 原文只作为普通文字展示，不提供英文单词点击释义。
- 用户使用中文输入法逐字输入汉字。
- 输入法拼音组合阶段不校验；候选词确认后，按输入的汉字逐个校验。
- 自动跳过空格、标点、数字和英文字母，只校验目标文本中的汉字。
- 输入错误的汉字显示红色，退格清除错误后才能继续。
- 正确率只按目标汉字数量和错误汉字次数计算。

### 英文模式

保持当前行为：

- 只校验英文字母。
- 忽略大小写。
- 自动跳过空格、标点、数字和其他非英文字母。
- 显示中文翻译、重新翻译按钮和单词释义。
